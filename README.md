## 프로젝트 소개

MuTune은 음악인을 위한 웹 기반 음악 편집 SaaS 서비스입니다.

현재는 악보(PDF) 처리 기능을 중심으로 개발을 진행하고 있으며,  
웹 환경에서 변환된 MusicXML 데이터를 렌더링하고  
사용자가 직관적으로 확인할 수 있도록 UI를 구성하고 있습니다.

단순 화면 구현이 아니라,

- 인증 상태에 따른 UI 제어
- 토큰 기반 API 통신 구조
- 악보 렌더링 처리
- 반응형 레이아웃 설계

를 고려하여 설계했습니다.

---

## Tech Stack

- React
- Vite
- Tailwind CSS
- Axios
- JWT 기반 인증 연동
- Verovio (MusicXML 렌더링)

---

## 주요 기능

### 1. 랜딩 페이지

- Full screen 구조
- CTA 중심 설계
- 로그인 / 회원가입 유도
- 서비스 방향성 전달 UI 구성

---

### 2. 로그인 및 인증 처리

- JWT 기반 인증 구조
- Access Token 저장 후 API 요청 시 Authorization Header에 자동 포함
- 로그인 상태에 따른 헤더 UI 분기 처리
- 인증이 필요한 페이지 접근 제어

백엔드의 Spring Security + JWT 구조와 연동하여  
Stateless 인증 흐름을 구현했습니다.

---

### 3. 악보 렌더링

- 백엔드에서 생성된 MusicXML 데이터를 수신
- Verovio 라이브러리를 사용하여 브라우저에서 악보 렌더링
- XML → SVG 변환 구조 적용
- 화면 리렌더링 시 상태 기반 재호출 처리

단순 이미지 출력이 아닌  
동적 MusicXML 기반 렌더링 구조를 사용했습니다.

---

### 4. API 통신 구조

- Axios 인스턴스 분리
- 공통 Base URL 관리
- 요청 인터셉터를 통한 JWT 자동 삽입
- 에러 응답 처리 로직 분리

백엔드 구조와 맞춰 확장 가능하도록 설계했습니다.

---

- components: 재사용 가능한 UI 컴포넌트
- pages: 라우팅 단위 페이지
- api: Axios 설정 및 API 요청 분리
- hooks: 상태 로직 분리
- assets: 이미지 및 정적 파일

기능 단위로 분리하여 유지보수성을 고려했습니다.

---

## 상태 관리 방식

- useState / useEffect 기반 상태 관리
- 로그인 상태 전역 관리
- API 응답 기반 조건부 렌더링 처리
- 렌더링과 데이터 로직 분리 구조 유지

대규모 상태 관리 라이브러리 없이  
현재 규모에 맞는 구조로 설계했습니다.

## 프로젝트 목적

이 프론트엔드 프로젝트는 단순 화면 구현이 아니라,

- 백엔드와의 인증 흐름 이해
- MusicXML 데이터 렌더링 처리 경험
- API 기반 상태 제어
- SaaS 구조를 고려한 UI 설계

를 직접 경험하기 위해 진행했습니다.

현재는 악보 기능 중심으로 구현되어 있으며,  
장기적으로 음악 제작 전체 흐름을 웹에서 해결할 수 있는  
플랫폼으로 확장하는 것을 목표로 하고 있습니다.
